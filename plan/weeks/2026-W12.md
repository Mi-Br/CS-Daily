## Reading (tonight Sat + commute Mon)

Do in this order:

**1. Effective Go — Composite Literals (10 min)**  
https://go.dev/doc/effective_go#composite_literals  
How Go authors think about grouping data. Read it twice.

**2. Data semantics in Go — Bill Kennedy (20 min)**  
https://www.ardanlabs.com/blog/2017/06/design-philosophy-on-data-and-semantics.html  
The core idea: *design your data first, your code follows from it.*  
Question to ask before writing any code: "what does my data look like?"

**3. JSON in Go — Go by Example (10 min)**  
https://gobyexample.com/json  
`json.Marshal`, `json.Unmarshal`, struct tags. Type it out, don't just read.

**4. Struct embedding (5 min)**  
https://gobyexample.com/struct-embedding  
How `json:"name"` works and why.

## Weekend Project — `todo` CLI

**Assigned:** 2026-03-22 (Sunday)  
**Estimated time:** 2-3h

### What to build

A CLI task manager. From scratch.

```
todo add "Buy groceries"   → adds a task
todo list                  → shows all tasks
todo done 1                → marks task 1 complete
todo delete 1              → removes a task
```

Tasks **persist between runs** — saved to a JSON file.

### Done when
- [ ] All 4 commands work
- [ ] Data survives closing and reopening the terminal
- [ ] Graceful errors (e.g. `todo done 99` when task 99 doesn't exist)
- [ ] `go vet` passes clean

### What to figure out yourself (the point of the exercise)
- What structs do you need? What fields?
- How do you store tasks in memory — slice or map?
- Where does save/load logic live? On what type?
- How do you parse `os.Args`?

### OK to google
- `os.Args` — how to read CLI arguments
- `os.ReadFile` / `os.WriteFile` — file I/O
- `json.Marshal` / `json.Unmarshal` (read the article first)
- `fmt.Fprintf(os.Stderr, ...)` — printing errors

### NOT OK to google
- "Go todo CLI tutorial" — you'll copy someone's struct design. That's the whole point.
